## What This Is

A single-file PWA for managing Daggerheart RPG character sheets. All logic lives in `index.html` (~2400 lines): embedded CSS, HTML markup, and vanilla JS. No build step, no package manager, no TypeScript. Firebase 10.12.0 is loaded via CDN for auth and Firestore sync. UI language is French.

## Running Locally

Open `index.html` directly in a browser. No server needed. Add `?debug` to the URL (or triple-tap the app title) to reveal the debug panel which logs Firebase and auth operations.

## Architecture

**Single global script block** organized into functional sections:

- **Constants & defaults** (top of script): trait keys, classâ†’subclass map, hardcoded `THELKRI_DEF` default character
- **State**: three globals â€” `chars[]` (all character definitions), `charDef` (current char being tracked), `state` (live session state)
- **Persistence layers** (priority order): in-memory â†’ localStorage â†’ Firestore. Saves are debounced 1.5s before hitting Firestore
- **Three screens**: `#screen-list` (character grid), `#screen-form` (create/edit), `#screen-tracker` (live play). Switch via `showScreen(name)`
- **Tracker HTML**: entirely regenerated by `buildTrackerHTML(def)` each time a character is opened â€” no incremental DOM updates, just a full rebuild
- **Event handling**: delegated listeners set up in `initTrackerEvents()` after each tracker rebuild
- **Dice roller**: `rollDuality(mod, traitName)` â€” 2d12 + modifier, with advantage/disadvantage, experience bonuses, and persistent roll history in localStorage
- **Firebase**: `initFirebase()` sets up auth + Firestore persistence. `loginWithGoogle()` uses popup-first with redirect fallback for mobile. Firestore paths: `users/{uid}.chars` (character list), `users/{uid}/states/{charId}` (session state)

## Key Patterns

- **Combo selects**: many dropdowns pair a `<select>` with a hidden `<input>` for custom text â€” use `readCombo()`/`setCombo()` helpers, not raw `.value`
- **Dynamic form rows**: weapons, experiences, features, domain cards, toggles, and charges are built by `add*Row()` functions and read back by iterating `querySelectorAll`
- **`mergeState(saved, def)`**: called when opening a tracker â€” clamps HP/Stress/Hope within current maxima, preserves layout preferences, fills in missing keys from defaults
- **`applyProficiency(formula, prof)`**: auto-scales dice in damage formulas (e.g. `d10+6` with prof 2 â†’ `2d10+6`) unless the weapon is marked `effective: true`
- **Mobile auth**: uses `signInWithPopup` first; on `auth/popup-blocked` or `auth/popup-cancelled` errors, falls back to `signInWithRedirect` and reads result on next load via `getRedirectResult`
